去年还像幻想，今年已经能开始半自动设计了。

`Open Pencil` 可以帮你先做更自动的第一轮搭建，也可以做更稳的半自动协作：先让 agent 跑握手、读画布、批量改版，你再自己补字体、拖图片、换素材、修细节。它不是单独一个神奇按钮， 而是要和 `takeaway-skill`、参考数据、以及你本地装好的依赖一起联动，才会越来越像你在我自媒体里看到的效果。

它真正帮你做到什么

这不是“看起来很酷但其实跑不起来”的空壳。它已经把 `Pencil` 这一层真正能复用的执行纪律开出来了。

全自动用嘴做设计 不用手写复杂提示词，直接用嘴语音输入也能做第一轮设计，默认更建议语音输入。
任何中间环节都能半自动接管 做一半也能继续改文字、改图片、加新元素，不需要整页推倒重来。
先握手，再开工 先测工具、先读画布、先确认能截图，不让人一上来就蒙着改。
批量执行，不碎修 改动更像一轮一轮执行，而不是随手点一点、歪一点、再补一点。
锁版后只点修 页面已经手工调过时，自动切换到更稳的锁版点修思路。

它不是什么

先把预期说清楚，别人反而更容易用对。

不是官方 Pencil 这里不提供官方本体，也不镜像、不再分发官方 MCP。
不是一键魔法包 它不能脱离 `takeaway-skill`、参考数据和本地依赖，独立复现我最强演示效果。
不是内部完整版 公开的是执行层，不是我内部那套已经练了很多轮的增强工作流。

为什么我自媒体里的版本看起来更强

因为我演示时，本来就不是只靠一个 public skill 在工作。

01 `takeaway-skill` 先判断什么值得学、什么不能直接抄。
02 `Open Pencil` 负责握手、读画布、改动执行、截图复核。
03 本地化依赖 我自己的脚本、运行环境和安装好的工具也在一起工作。
04 更多参考数据 internal 里积累的截图、录屏、蒸馏卡和中间资产也会继续放大效果。

如何载入和使用

不要一上来就让 agent 干整页设计，也不要让它跑一堆安装命令。先载入 Skill，再单独做握手测试。

01 先从官方渠道在本机安装 Pencil，并确认 MCP 工具已经出现。
02 这个仓库不是 npm / pip 包，也不是 App 安装器，不需要跑长安装脚本。
03 打开这个仓库后，先读 `skill/SKILL.md`，确认 Skill 可读取后就停。
04 再单独让本地 agent 做握手测试，确认能读画布、能截图、能调用工具。
05 握手通过后，再结合 `takeaway-skill` 和参考数据正式执行。

如果失败了，通常说明什么

先把这句话写清楚很重要：有些功能跑不起来，不一定是这个 skill 不好，更常见的是你还缺依赖。

官方 Pencil 没装好 或者 Pencil 没启动，所以工具链根本没连上。
MCP 工具没出现 客户端里还没看到工具，说明这轮连基础握手都没完成。
缺少上游和数据 没接上 `takeaway-skill`，或者没给够截图、录屏、蒸馏卡，执行层自然不会像演示里那么强。

默认使用流

真正常用的路线不是“只读一个 skill 文件”，而是一整条链路一起工作。

01 先装好官方 `Pencil`。
02 用 `takeaway-skill` 先判断参考里什么值得拿。
03 准备好截图、录屏、蒸馏卡等参考数据。
04 再用 `Open Pencil` 跑握手、读画布、改动执行和截图复核。
