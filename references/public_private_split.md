# Public / Private Split

This repository is designed around a three-layer model.

## Layer 1: Public framework

Safe to open:

- docs
- workflow instructions
- release checklist
- public skill file
- simple landing page
- safe starter shell

This layer is good for:

- visibility
- onboarding
- credibility
- community discovery

## Layer 2: Public examples

Can open only after review:

- neutral example `.pen` files
- public-safe screenshots
- fake or placeholder copy
- generic UI kits that do not leak private style systems

This layer needs a final privacy and license pass before release.

## Layer 3: Private moat

Keep private:

- advanced prompt choreography
- private benchmark packs
- paid workflow combinations
- internal style guides
- private case libraries
- partner or client materials
- third-party study archives

Rule of thumb:

If someone can fork this repo and immediately reproduce most of your real advantage, that piece does not belong in the public layer.

## Business framing

Public should help with:

- visibility
- trust
- inbound opportunities

Private should keep:

- conversion strength
- differentiation
- repeatable premium output
