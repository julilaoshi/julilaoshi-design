# 参与贡献

谢谢你愿意参与。

不用是大神才可以贡献。如果你真的用过这个项目，发现哪里不清楚、跑不通、绕得慌，这些反馈就很有价值。

## 欢迎什么样的贡献

你可以为下面这些情况提 issue 或 pull request：

- README 说明不清楚
- 链接、图片、视频或 demo 失效
- 在 Codex、Claude Code 或其他 coding agent 里加载失败
- 某个流程步骤太绕
- 提示词可以更安全、更清楚
- 有公开安全的案例或使用场景可以补充
- 中英文表达、文档说明可以改得更自然
- 发现小 bug，并且能提供足够上下文

## 请不要提交什么

请不要提交这些内容：

- 你电脑里的私有路径
- 私人聊天记录
- 客户、学生、用户的私有材料
- 没有授权的图片、字体、视频、网页素材
- 对别人作品的表层照搬
- 刷 star、假数据、推广垃圾信息
- 内部提示词链、私有工作流、未公开细节
- 和项目方向无关的大功能

这个项目欢迎“把参考变成方法”，不欢迎“换皮复制”。

## 如何提 Issue

提 issue 时，最好写清楚：

- 你想做什么
- 实际发生了什么
- 你本来期待它怎样
- 如果相关，说明你的环境
- 如果要放截图或链接，请确认它们可以公开

不用写得像论文。  
能让我快速判断问题在哪里，就很好。

## 如何提 Pull Request

小的文档修正，欢迎直接提 PR。

如果你想改工作流核心逻辑，建议先开 issue 说一下使用场景。这样可以避免公开版越来越臃肿，也能保持项目安全、轻量、好维护。

如果 PR 会改变核心流程，请说明：

- 它解决了什么问题
- 会不会影响老用户
- 有没有隐私、版权或兼容性风险

## 关于公开安全案例

欢迎补案例，但案例必须是公开安全的。

比较适合的案例包括：

- 你自己的 demo
- 你自己的截图
- 公版或授权清楚的素材
- 抽象 mockup
- 重新做过、没有照搬表层风格的示例

请不要把私人项目、私人信息、电脑路径、未授权视觉素材放进来。

## 维护者说明

有些内部工作流、私有素材、高级实验、未公开内容，不会放在这里。这是有意保留边界。

公开版最重要的不是“什么都有”，而是能看懂、能加载、能安全使用。

如果这个项目对你有用，欢迎点个 star 支持一下。  
但不要刷，不要假互动，不要机器人。真实一点，慢慢来。
