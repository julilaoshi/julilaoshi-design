# Maintenance

`julilaoshi-design` is maintained as a lightweight public design workflow.

The goal is to keep the public version readable, useful, and safe for GitHub users, while keeping private workflows, private assets, and advanced internal experiments outside this repository.

## What We Maintain

We actively maintain:

- README clarity
- first-run loading instructions
- `Pencil MCP Mode` vs `Web Editor Mode` explanations
- Windows-friendly browser-first usage path
- right-side Web Editor panel behavior
- basic editor actions such as hide, delete, duplicate, align, done, and export
- public-safe examples and walkthrough materials
- broken links, missing images, and unclear demo instructions
- contribution and issue guidance
- privacy, copyright, and public/private boundary wording

## User Feedback Sources

Feedback may come from GitHub issues, social media comments, video comments, direct usage reports, or early user testing.

If feedback comes from outside GitHub, it should not be presented as a GitHub issue unless an issue is actually opened. It can still be summarized in maintenance notes, changelogs, or future GitHub issues as external user feedback.

## Known Project Boundaries

This repository does not maintain:

- the official Pencil app itself
- official Pencil installation or platform availability
- official Pencil MCP server behavior
- private prompt chains
- private design assets
- private client, student, or user materials
- advanced internal workflows that are not public-safe
- large unrelated feature requests

## Platform Notes

`Pencil MCP Mode` requires official Pencil and MCP access on the user's local machine.

`Web Editor Mode` is the recommended public path for users who cannot use official Pencil, including many Windows users and beginners who want to start directly in the browser.

## Maintenance Priorities

Current public maintenance priorities:

1. Keep the first-run choice simple.
2. Make the difference between Pencil MCP and Web Editor easy to understand.
3. Improve the right-side Web Editor panel.
4. Keep the public version lightweight and public-safe.
5. Avoid leaking internal workflows, private assets, or unauthorized visual materials.

## Feature Direction

Future improvements may include:

- clearer right-side panel controls
- better layer editing behavior
- more public-safe examples
- improved export guidance
- better beginner onboarding
- clearer walkthrough materials

Large features should be discussed in an issue before implementation.
